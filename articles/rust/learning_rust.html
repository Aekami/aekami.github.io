Starting to learn Rust

Why

I was already familiar with high level languages (Java, C#), and with very high level langages (Js), and had some basic understanding of low level langages (C, Assembly). But I had never done any actual project in a low level language, only small programs. The most advanced program(s) being a client-server app in C (not using much premade library, reading them sockets raw >:) ). The "frontend" being the CLI. Also, I wasn't sold on C. I'd likely have prefered C++, but my school was doing C, so C it was.

So now that I'm free to pick whatever I want, I went on to look for suitable candidates. My choice was among the following :

  • C++ - Of course, why not just learn C++ ? It's tested, it's robust, it has very little to prove.
  • Go - Great for concurency, and I was told easier to learn. But I also wanted to be forced to learn the underlying workings of a program (which includes managing memory correctly), and go being a gb-collected language suited that less.
  • Zig - Lots of hype around this one. What made me want to potentially learn it is actually a talk about how one of the Zig developper optimized the language, compiler, etc.
  • Rust - If Zig is hyped, Rust is overhyped. Set out to replace C/C++ over time, and getting adopted more and more, in Google, in Microsoft, etc. It essentially checks my prerequisites of wanting to be forced to understand what's happening, and even better there's a (helpful?) compiler to tell me when I mess up.
